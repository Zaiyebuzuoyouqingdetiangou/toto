# RabbitMirror 0.31.92

本版以 0.31.91 为底座，仅增强智能交互急救。

- 修复宿主在 DOM 渲染前删除 `onclick` 后，0.31.91 无法再读取伪交互程序的问题。
- 急救器会从当前聊天消息的原始 HTML 中回读安全可解析的 `document.getElementById(...)` 样式与文字赋值。
- 通过兔子镜标题与元素子路径映射回当前渲染节点，目标仍严格限制在当前兔子镜内部。
- 不执行原 JavaScript，不使用 `eval`；包含未知语句时仍然拒绝。
- Prompt、Visual Scenery、主题库、设置键、代码块急救均未修改。

详细变更见 `README-0.31.92.md`。

## 0.32.5 正式修复版
- 移除 0.32.4 的临时可视化诊断面板。
- 新增 label 内单块隐藏内容急救：排除隐藏 input，识别同一 label 中真正的隐藏正文。
- 勾选时同步修复 opacity、visibility、transform、display、height、max-height 与 overflow 裁切；取消后恢复原样。
- 兼容宿主删除 onchange 的情况，不执行模型生成的 JavaScript。
- Prompt 与 0.32.4 保持一致，不增加模型输入规则。

## 0.32.6 正式内置诊断版
- 保留 0.32.5 的 label 内隐藏内容急救。
- 新增默认关闭的“交互诊断面板（反馈用）”开关。
- 开启后在每个兔子镜底部显示版本、命中路线、事件、控件状态、隐藏候选、实际样式与裁切信息，并可一键复制。
- 关闭开关后移除诊断面板与诊断事件监听。
- 诊断只在浏览器本地运行，正文仅显示截断片段，不上传数据。
- Prompt、配色节奏、折叠冷却与 Visual Scenery 规则均未修改。
