## 0.33.1 TEST｜🐇 小小维修兔 v1.1 合一版

### 真正合并旧急救入口

- 设置面板不再显示“代码块急救模式”“智能交互急救”及“修复单条纯文字兔子镜”三个旧入口。
- 统一保留一个“🐇 启用小小维修兔”开关；旧用户若曾开启任一旧急救，升级时会迁移为维修兔启用，旧全局扫描随后强制关闭。
- 代码块／纯文字恢复、源码与损坏 SVG 保主体、交互状态恢复和触屏 Hover 兜底，均归入维修兔内部模块。
- RabbitMirror 全链路诊断继续独立保留，用于红灯、未渲染代码块或纯文字源码等无法挂载标题维修兔的情况。

### 严格单条维护

- 每条已渲染兔子镜标题后各自显示 `🐇⚪`，一只维修兔只绑定当前这一条兔子镜。
- 点击白灯只巡逻当前条目；黄灯再次点击只维修当前条目；红灯只生成当前条目的全链路诊断。
- 源码重绘使用当前消息的临时副本，不写回 `mes`、swipe、`display_text` 或聊天存档。
- 含思维包裹或独立显示源时拒绝整条重绘，避免绕过显示正则。
- 不再把“尚未建立隔离”本身当作故障证据，避免正常交互被误判为黄灯。
- 设计原则：**没有证据证明它坏了，就不要碰它；永远不影响其他兔子镜。**

### 状态

- `🐇⚪`：待巡逻。
- `🐇🟢`：未发现高置信异常。
- `🐇🟡`：当前条目命中已有安全修复能力。
- `🐇🔴`：无法安全判断，转交当前条目的全链路诊断。

维修兔与诊断均为本地运行逻辑，不进入 Prompt，不增加模型 token。

## 0.32.76 TEST｜Menu QR v2.1 动态菜单版

### 本版母本／抽取池一致性审计

- 主题母本 101 项与主题抽取池 101 项逐项一致，无遗漏、无重复 ID。
- 展现形式母本 177 项与展现形式抽取池 177 项逐项一致，无遗漏、无重复 ID。
- 将“光点颜色解码表”明确为“萤火虫栖息地”的说明表，不再伪装成独立可抽取条目。
- 修正 `1.8 OA系统审批流` 的 Markdown 粗体闭合。
- `5.3.2.2` 的原港式命名改为大陆语境更自然的 `传统寺庙灵签解读`，并同步更新母本、结构化池与 Menu QR v2.1。

本版以 0.32.74 TEST 为唯一基线。

### 点菜 QR 重构
- `RabbitMirror Menu QR v2.1` 与插件本体继续使用独立版本号。
- QR 节点由 360 个缩减为 7 个，避免快捷回复管理页渲染数百行而卡顿。
- 101 条主题元素与 177 条展现形式全部保留在两个动态 STscript 菜单中。
- 保留三级／多级浏览、父级直接选择、自定义点菜、补充要求、生成一次性订单和清空功能。
- 点菜结果只写入输入框，不自动发送。
- 一次性订单等待下一次实际生成的兔子镜，采用一次后失效。

### Token 与加载
- QR JSON 仍是插件内静态资源，仅在用户点击下载按钮时下载。
- 不注入 Prompt、聊天记录或模型上下文，不增加聊天 token。
- 单个 QR 脚本文本较长，但管理页只渲染 7 个节点。

### 保留功能
- 101 条主题元素、177 条展现形式。
- SVG Data URI 保主体急救与结构截断诊断。
- data-active 同组状态急救。
- RabbitMirror 全链路诊断、源码恢复、思维链隔离、可读性规则和暗色冷却。
