# RabbitMirror 0.33.36 TEST｜缓存安全接管修复版

## 本次修复

- 修复扩展列表已显示新版，但设置面板与标题按钮仍由旧版运行代码生成的问题。
- manifest 改用全新的版本化入口与 CSS 文件名；内部 RabbitMirror ESM 模块也使用版本化请求地址，避免 SillyTavern 热更新继续复用旧模块缓存。
- 新运行时会主动移除旧版或重复的设置面板，并重新挂载 0.33.36 设置 UI。
- 每条兔子镜标题后的维修兔与挨打猫都带真实运行版本标记；发现旧按钮时会先替换，再安装当前版本入口。
- 挨打猫按钮改为真实文本内容，不再依赖新 CSS 的伪元素才能显示黑猫。
- 新增运行时清理接口，为后续更新时释放观察器、定时器和本插件 UI。
- 小小维修兔更新为 v1.29；挨打猫保持 v1.0。

## 保持不变

- 继承 0.33.35 的三类交互修复与“文字被裁切或显示不全”维修功能。
- Prompt、美化规则、主题池、展现形式池、Visual Scenery、挨打猫反馈文案与 `data/safetyPatch.js` 均未改动。
- 用户未选择挨打猫反馈时，不会向模型 Prompt 追加任何内容。
