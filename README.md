# 兔子洞小剧场 / Rabbit Hole Theater

SillyTavern 第三方扩展：自动在每轮主回复正文最底部追加【兔子洞】小剧场。

## 当前逻辑

- 执行规则：每轮完整注入。
- 通用执行规则：每轮注入精简核心版。
- 主题元素：先抽结构化 ID，再到 `data/raw/` 母本中检索对应完整描述，只注入抽中的片段。
- 展现形式：先抽结构化 ID，再到 `data/raw/` 母本中检索对应完整描述，只注入抽中的片段。
- 防重复：由插件代码执行。
- 安全补丁：不注入。
- Toto v0.15：只显示在插件设置界面，不生成进小剧场。
- Visual Scenery 动态渐变模式：勾选后，本轮展现形式锁定为 `10.2 可视化小剧场` + `10.2.2 Visual Scenery`，并在内部注入父级与子项说明；设置界面不显示详细灵感说明。
- UI 自查优化：勾选后，每轮注入去模板化和 UI 差异自查要求，减少相似黑框/普通记录卡。

## 正文指令优先

设置界面勾选「正文指令优先」后，插件会读取最后一条用户消息中的兔子洞指令。

可用写法示例：

```txt
兔子洞：法甜剖面图
```

```txt
兔子洞主题：宫斗
兔子洞展现形式：OA审批流
```

```txt
兔子洞格式：短信体
```

```txt
小剧场想看法甜剖面图
```

```txt
下一个小剧场想看Visual Scenery
```

```txt
兔子洞关闭
```

规则：

- 用户指定主题或展现形式时，插件优先使用指定内容。
- 用户只指定一部分时，未指定的部分继续随机补足。
- 匹配方式支持 ID、标题、摘要、原始片段的模糊匹配。
- 写“兔子洞关闭 / 不要兔子洞 / 本轮不要兔子洞”时，本轮会跳过兔子洞注入。

## 安装

仓库根目录需要直接包含：

```txt
manifest.json
index.js
style.css
README.md
data/
src/
```

在 SillyTavern 中通过 Extensions → Install Extension 粘贴仓库地址安装。
