# RabbitMirror 0.33.47 TEST｜文生图画框根节点修复＋2D 双模式＋挨打猫语言反馈

## 0.33.47 修复

- 文生图不再只识别 `<toto data-rabbit-mirror="true">` 外壳；SillyTavern 移除未知标签、只保留兔子镜 `<details>` 时也能创建画框。
- 新增独立 DOM 监听，不再只依赖生成事件与视觉扫描时序。
- 免费模式和自定义 API 模式都在请求前创建“绘制中”画框；失败时保留可点击的“绘图失败”画框并显示原因。
- 插件启动和开启开关时只把旧消息作为基线，不会倒查旧兔子镜并产生请求。
- 同一条消息／同一 swipe 仍最多发起一次图片请求。

## 0.33.46 修复

- 活跃的挨打猫反馈直接追加到 RabbitMirror 主隐藏 Prompt 的最末尾，不再依赖独立 Prompt 的模型侧优先级。
- 未选择反馈时不追加任何字符，基础 RabbitMirror Prompt 保持不变。
- 「一直说外语」增加最终可见文字检查：标题、按钮、标签、状态、提示、角标、占位文字与装饰词不得残留不必要外语。
- 语言反馈不再在任何成功兔子镜上直接扣轮数；只有可见语言检查通过才消耗。
- 若下一轮仍检测到英语，反馈保持原轮数，并在挨打猫状态中显示“上一轮未落实，反馈继续保留”。
- 保留 0.33.45 的文生图触发、免费直链、移动端 DOM 等待和自定义 API 代理回退修复。
