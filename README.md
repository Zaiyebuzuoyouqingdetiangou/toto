# RabbitMirror 0.33.37 TEST｜挨打猫送达验证版

## 本次更新

- 挨打猫更新为 v1.1；小小维修兔保持 v1.29。
- 反馈优先保存到当前聊天的 `chatMetadata`，并保留 `localStorage` 兼容兜底。
- 用户保存反馈时，立即注册一条独立的隐藏扩展 Prompt；下一次生成拦截器会再次读取、刷新并记录送达状态。
- 切换聊天时会自动同步或清除独立反馈 Prompt，避免旧聊天的反馈串入新聊天。
- 自定义反馈完整保留用户原话；要求最终兔子镜清晰落实，但禁止把反馈原话、挨打猫说明或执行过程显示到聊天正文与兔子镜成品中。
- “配色不喜欢、结构太模板、整体不好看、交互太简单”等预设反馈，会附带被点击作品的粗粒度浏览器视觉指纹；不会发送上一条小剧场正文。
- 挨打猫菜单新增送达状态与上一轮送达回执，区分“等待生成、拦截器已读取、成功生成后已消耗”。

## 保持不变

- 以 0.33.36 功能代码为基线，保留缓存安全入口、交互修复、文字裁切维修和标题按钮安装链。
- 主题池、展现形式池、Visual Scenery、固定美化规则和 `data/safetyPatch.js` 均未改动。
- 没有有效反馈时，独立反馈 Prompt 为空；基础 RabbitMirror Prompt 与 0.33.36 的无反馈结果逐字一致。
- 挨打猫不会写入聊天发送框、`mes`、swipe 或 `display_text`。
