# RabbitMirror 0.33.42 TEST｜文字承载修复＋2D 文生图双模式

## 本版修复

- 修复维修兔“文字被裁切或显示不全”模式把结构外壳统一改成 `overflow:visible`，导致翻页、卡片、相框等内容漏出所属区域的问题。
- 对 `position:absolute + height:100%` 承载可增长正文的状态页，优先让状态页与定位父级按内容增高；视觉外壳继续保留原裁切边界。
- 普通文字节点的换行、行数解除与宽度修复仍保留；只有文字载体自身被截断时才解除其 overflow。
- 复杂交互视觉核心补充：可增长正文不得只依赖 absolute、固定高度或 height:100%；父级须自然增高、预留空间，或提供媒介内部滚动／分页。
- 保留 0.33.40 的跨层伪类交互修复。

## 文生图（测试版）

- 文生图设置固定放在 RabbitMirror 设置 UI 最下面。
- 总开关默认关闭；关闭时不构建图片 Prompt、不读取 CHAR／USER 外观、不调用图片接口、不生成画框。
- 两种生成方式：
  - 免费成图：使用内置测试成图地址，不需要用户填写 API。
  - 自定义 API：填写接口地址与 API Key，点击“连接并拉取模型”后选择模型。
- 两种模式都统一使用 2D 动漫画风。
- 文生图使用独立 Prompt，不进入 RabbitMirror 主 Prompt，也不写入聊天消息。
- 配图依据当前兔子镜可见内容，并读取可用的 CHAR／USER 外观描述作为人物锚点。
- 默认附加人体稳定、防乱码／UI／水印，以及非血腥、非暴力、非恐怖约束。
- 图片显示在兔子镜内容区右下方小画框，点击可查看大图；不新增标题按钮。
- 同一轮最多请求一次，失败后不自动无限重试。
