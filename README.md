# RabbitMirror 0.33.20 TEST｜展现形式优先・状态化交互强化版

## 0.33.20 状态化交互与维修兔空白分流修正

- 以用户提供并已修改完成的 `0.33.19 TEST` 实际 ZIP 为唯一开发基线；保留展现形式优先、0.32.56 审美回调、移动端排版、无 label 交互与 iOS 翻面修复。
- 删除 0.33.19 中较弱且重复的“至少一个交互变化／第二状态”描述，合并为紧凑的状态化完整交互链：对象→操作→可识别状态→内容或结构反馈→继续推进、分支、组合、切换或返回。
- 每轮仍强制真实交互，但交互形态、规模和阶段必须由本轮媒介结构、使用/阅读逻辑与叙事产生，不把 checkbox、翻面、弹窗、按钮组或标签页作为默认换皮骨架。
- 非天然一次性媒介中，首次操作不得耗尽全部体验；伪选项、仅样式变化、无关堆叠或一次显隐即结束不算完整交互。
- Visual Scenery 恢复“双硬要求”：完整、持续、肉眼可见的 CSS 动画与完整状态化交互链必须同时成立，二者不能互相替代。
- 小小维修兔 v1.16 新增空白主体判定：展开后没有显示兔子镜主体、但原始源码仍存在时，自动判断和菜单建议优先选择“📄 显示代码或纯文字”，不再被丢失控件证据误导为交互修复。
- 全链路诊断新增“展开后主体缺失／主体子节点”证据；仍只维护当前消息、当前兔子镜，不写回 `mes`、swipe 或 `display_text`。

## 0.33.19 展现形式优先与审美回调

- 以用户提供的 `0.33.18 TEST` 实际 ZIP 为唯一功能基线，完整保留无 label 交互、iOS 3D 翻面和小小维修兔修复链。
- 以 `0.32.56 TEST` 仅作为审美与 Prompt 压力对照，不用旧代码覆盖新版功能。
- 将 0.33.18 中“完整交互链＋第二层体验”的刚性压力回调为 0.32.56 的较轻交互要求，减少为完成指标硬塞 checkbox、翻面和隐藏层的倾向。
- 删除“本轮独有色彩与光感／主动改变色彩关系”，避免为了求新强行加入霓虹、光晕、高饱和色或不属于媒介的配色。
- 将展现形式确立为最高视觉依据：布局、轮廓、材质、字体、配色、光影、动效与交互均须从媒介本体推导。
- 明确允许报告、审批流、属性面板、排行榜等媒介保留原生信息结构；禁止的是无关的通用卡片壳，不是媒介本身的信息组织。
- 视觉克制不设固定数量模板，只限制无意义堆叠，同时禁止以“克制”为由削弱媒介天然需要的丰富结构。
- 新增紧凑的移动端排版与双面结构规则：以 360～430px 为首要画布，避免正文处于持续 transform 阅读层，正反面共用定位平面且仅中间层负责翻转。
- 未修改主题池、展现形式池、Menu QR、维修模块行为、聊天原文写回策略或本地急救库。
- 以同一组测试参数测得基础注入 Prompt 从 3193 字符降至 3058 字符，未增加模型 token。

## 0.33.18 无 label 实际切换链与 iOS 分离样式翻面修复

- 无 `label` checkbox/radio 宿主兜底改为增量校准：即使第一次扫描时 `style` 尚未流入、结果为 0，后续维修也会重新扫描并真实安装，不再被空 WeakMap 永久短路。
- 宿主点击后记录 `before → intended → actual`，并在 0/60ms 二次核验；局部宿主由急救器接管时阻断同次旧点击链，避免一次点击重复切换两次。
- `:checked` 命中检查新增 CSSOM 递归，支持 `@media`／`@supports` 内规则及 `:has(input:checked)` 宿主状态。
- WebKit 3D 修复不再要求 `backface-visibility` 与 `rotateY` 位于同一个 `<style>`，可补齐分离 style 与流式晚到 style；重复执行不会重复堆叠前缀。
- 只复制原本已有的 `rotateY`、`backface-visibility`、`transform-style`、`perspective` 到对应 `-webkit-` 声明，不给整个容器额外添加 `rotateY`。
- 全链路诊断新增无 label 最后一次真实状态核验，以及 rotateY／backface／preserve-3d／perspective 的标准版与 WebKit 版对照证据。
- 不修改 Prompt、主题池、展现形式池、Menu QR，也不写回 `mes`、swipe 或 `display_text`。

## 0.33.17 iOS 无 label 交互与 3D 翻面修复

- 新增无 `label` 的 checkbox/radio 局部宿主点击兜底：只在当前控件确实存在有意义的 `:checked` 后置状态规则时启用，点击当前条目即可可靠切换状态。
- 针对 iOS Safari/WebView 补充 `-webkit-backface-visibility`、`-webkit-transform-style`、`-webkit-perspective` 与 `-webkit-transform`，修复 3D 翻面后文字镜像或正反两面同时显示。
- 维修兔“点了没有反应”会安装无 label 控件宿主路线；“样子不对”与“全部试试”会运行 iOS 3D 翻面兼容模块。
- 巡逻在 iOS 检测到 3D 翻面结构且尚未兼容时，会推荐“🎨 样子不对”。
- 全链路诊断新增“无label控件宿主”与“iOS 3D翻面兼容”计数。
- 不修改 Prompt、主题池、展现形式池、Menu QR，也不写回 `mes`、swipe 或 `display_text`。

## 0.33.13 直接源码重建与运行时残留引用修复

- 修复 `repairMaintenanceMessageSource()` 调用了不存在的 `getSelectedMessageSource()`，导致源码恢复入口在浏览器运行时中断的问题。
- 修复巡逻链调用了不存在的 `decodeHtmlEntitiesForRescue()`，但被 `try/catch` 静默跳过的问题。
- 遇到 CSS ERROR、TH-render 或高亮代码壳时，优先从当前消息的安全源码中只重建当前这一面兔子镜 DOM，不再把修复后的源码重新交给同一个 TH/Markdown 解析链。
- 直接重建前会继续执行 CSS 注释剥离、原始 SVG Data URI 百分号编码、状态 `#id` 兼容改写、逐镜 CSS class 隔离。
- 若消息使用独立 `display_text`，只从该显示源恢复，不回退到 `mes`/swipe，避免绕过显示正则。
- 含 thinking/reasoning 包裹、脚本、iframe、内联事件或 `javascript:` 的源码不会进入直接 DOM 重建。
- 修复后巡逻只在当前显示层仍有 CSS ERROR、代码壳、转义源码或纯文字候选时报告源码故障；原始聊天源码未写回不再造成永久黄灯。
- 维修模块记录新增 `sourceRepair`，可在后续诊断中确认源码恢复是否实际执行、采用了哪条理由。
- 未修改 Prompt、母本、抽取池、主题、展现形式或 Menu QR v2.1。

## 0.33.12 自动分流与 CSS 报错识别修复

- 维修兔菜单新增“自动判断并维修（推荐）”。
- “显示代码”与“显示纯文字”合并为同一条“源码显示异常”恢复路线，用户无需判断二者。
- 自动模式会优先识别原始源码被 CSS ERROR、TH-render 或高亮代码壳遮蔽的情况，并选择源码重建路线。
- 源码恢复路线统一执行代码块、纯文字、已渲染 details、CSS 注释边界与交互 ID 隔离模块。
- 扩大宿主 CSS ERROR 识别范围，不再只识别 `property missing ':'`；同时识别 `missing '}'` 等宿主报错。
- 全链路诊断新增 CSS 错误摘要与原始 CSS 注释计数。
- 当原始 CSS 含注释且宿主报错时，诊断会明确判断为注释边界中断；源码重建前会移除 CSS 注释。
- 保留旧 `code` / `plainText` 内部模式兼容，但新版菜单不再要求用户二选一。
- 未修改 Prompt、母本、抽取池、主题、展现形式或 Menu QR v2.1。

## 0.33.11 CSS 链恢复修复

- 修复维修兔从原始消息重绘时，`<style>` 内原始 SVG Data URI 仍可能触发宿主 CSS 解析错误的问题。
- 仅对明确包含原始 `<svg>` XML 的 quoted `data:image/svg+xml` 资源执行百分号编码；base64 与已编码资源保持不变。
- 将 CSS 规则中的简单 `#id` 状态选择器改写为等价的 `[id="..."]`，避开部分宿主／美化作用域解析链在状态选择器附近中断的问题。
- checkbox、radio、`label for` 与后续逐镜 ID 隔离语义保持不变。
- 全链路诊断新增“宿主 CSS 解析错误／原始未编码 SVG／原始 #ID 选择器”证据，并修正源码候选判断顺序。
- 未修改 Prompt、母本、抽取池、主题、展现形式或 Menu QR v2.1。
