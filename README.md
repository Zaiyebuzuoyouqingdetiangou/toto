# RabbitMirror 0.33.35 TEST｜交互状态与文字裁切修复版

## 0.33.35 修复内容

- 以 0.33.34 实际源码为唯一开发基线；挨打猫、原有美化规则、主题池、展现形式池、Visual Scenery 与 safetyPatch.js 保持不变。
- 每条已渲染兔子镜标题后新增独立的 🐈‍⬛ 挨打猫入口，提供：配色不喜欢、结构太模板、整体不好看、交互太简单、一直说外语、我要亲自骂。
- 反馈可选择只影响下一轮、接下来 3 轮或接下来 10 轮；每个聊天只保留一条当前反馈，新反馈会替换旧反馈，并可随时清除。
- “我要亲自骂”会保留用户原话，只做长度与控制字符清理；不会自动概括、扩写或套入开发者示例。
- 用户未选择反馈时，挨打猫不会向模型 Prompt 追加任何字符，原有美化规则与 token 消耗保持不变。
- 用户选择反馈后，反馈区块会在最终输出契约之前进入真实生成 Prompt；只有成功生成并渲染正式兔子镜后才扣除次数，失败生成、维修、诊断、重复渲染和用户交互不会消耗轮数。
- 修复扩展热重载后旧设置面板阻止新开关出现的问题；设置挂载稍晚时会自动重试，聊天 DOM 变化时也会补装维修兔与挨打猫标题入口。
- 修复隐藏 radio 由 label 兜底切换后，原始安全 onclick 只监听 input click、导致选中成功但结果区不显示的问题；现在安全动作也会跟随 change 状态执行。
- 加强 label 内隐藏正文的状态落地：label 兜底切换后立即执行对应显示路线，并保留延迟复核。
- 修正 classList.toggle 自变化交互的诊断误报；此类交互不再因没有“隐藏内容候选”被判定失败。
- 收紧触屏 Hover 报警：普通装饰性 Hover 不再压过已经成立的点击状态路线。
- 小小维修兔新增“🔤 文字被裁切或显示不全”，仅在用户点击后检查并解除固定高度、禁止换行、line-clamp 与溢出裁切。
- 挨打猫与小小维修兔的设置介绍继续保持一句话；小小维修兔更新为 v1.28。

## 保留的既有能力

0.33.33 及此前的通用 checked 选项状态修复、逐条维修兔、空白／源码恢复、SVG 保主体清洗、触摸状态恢复、跨父层 checked、伪元素状态切换、旧消息全选联动、伪交互识别等能力全部保留。
