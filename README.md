# RabbitMirror 0.33.39 TEST｜随兔子镜生成配图测试版

- 以 RabbitMirror 0.33.38 TEST 实际源码为功能基线。
- 保留正常根目录：`index.js`、`style.css`、`manifest.json`，不新增版本化根文件。
- 新增「随兔子镜生成配图（测试版）」总开关；默认关闭。
- 关闭时不构建文生图 Prompt、不读取 CHAR／USER 外观、不调用图片接口、不生成画框。
- 开启后，仅在本轮兔子镜正式生成成功时调用一次用户填写的 OpenAI 兼容图片 API。
- 文生图使用独立 Prompt，不进入 RabbitMirror 主 Prompt，也不写入聊天消息。
- 配图依据兔子镜当前可见内容，并读取可用的 CHAR／USER 外观描述作为人物锚点。
- 论坛、聊天、档案、报告等信息媒介会转换为场景，不直接生成网页截图。
- 默认附加人体稳定、防乱码／UI／水印，以及非血腥、非暴力、非恐怖约束。
- 图片以内容区右下方小画框显示，点击可查看大图；不会新增标题按钮。
- 同一轮多次宿主事件只允许一次图片请求，失败后不自动无限重试。
- 按用户确认补入「交互产生、替换或推进后的主要正文与关键反馈……」承载规则；除此之外，原 RabbitMirror 主 Prompt 保持不变。

## 测试版接口格式

填写完整的 OpenAI 兼容图片生成接口地址，例如以 `/v1/images/generations` 结尾的地址，并填写接口要求的模型名称。请求从浏览器发出，接口需要允许跨域访问。普通文生图网站首页不能直接用于自动生成。
