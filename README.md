# RabbitMirror 0.33.65 TEST｜挨打猫多选反馈

## 0.33.65 升级

- 挨打猫升级为多选反馈，可同时选择配色、结构、整体、交互、外语与其他说明。
- 选择项可反复勾选或取消，统一提交后再选择影响轮数。
- “其他”仅在选中后显示输入框；空内容不能提交。
- 关闭或取消时不保存草稿，不会影响后续生成。
- 多项反馈会合并进入同一条提示，要求同时协调落实，而非只执行其中一项。
- 保留旧版单项反馈记录兼容读取。

## 0.33.65 修复

- 新增“频道／画面双向切换”专项维修：识别两个独立 checkbox 分别承担“进入下一画面”和“返回主画面”，且两组 `:has(...:checked)` 规则互为反向显示的高置信结构。
- 解决反馈层带内联 `display:none` 时，普通 CSS `display:block` 无法覆盖而导致“主画面已隐藏、下一画面仍不可见”的空屏问题；维修后的状态显示使用当前局部 DOM 的受限 `display !important` 同步。
- 点击进入后仅保持前进控件选中；点击返回会同时清除前进与返回控件，恢复默认主画面，因此可反复执行“进入 → 返回 → 再进入”，不会在第二次操作时被残留 checked 锁死。
- 不执行模型生成的 JavaScript，不猜测缺失内容；只接管两个 checkbox 各自位于对应状态层内、并在 CSS 中明确互换同一对内容层的结构。
- 新增“电视／终端屏幕保形”识别：电视机、CRT、监控屏和终端外壳包含真实状态画面时，不再被通用手机 Flex/Fit/Break-text 维修批量改写外框、屏幕比例、扫描线、天线、旋钮及内部画面骨架。
- 全链路诊断新增“双向画面切换”与“电视／终端屏幕保形”计数；维修前会把这类状态锁死报告为交互问题，不再优先推荐无关的“排版不适配／内容显示不全”。
- 未修改母本库、Prompt、配色或展现形式规则；本次仅修当前兔子镜 DOM 的状态往返与手机端视觉保形。

## 0.33.63 保留修复

- 新增固定 checked 赋值的安全恢复：宿主删除 `onclick="document.getElementById('...').checked = true/false"` 后，维修兔只解析当前兔子镜内真实 checkbox/radio 与固定布尔值，不执行原始 JavaScript。
- 适用于甜品剖面、材质分层、时间切片等“点按一层切换 radio，再由 checked 展开说明”的结构；点击其他层时会切换单选状态并撤回上一层的内联兜底样式。
- 新增“剖面／分层保形”识别：纵向分层宿主及其内部色块、连接线、文字层不再被通用手机 Flex 维修加入 `flex-wrap`、单列堆叠、普遍 `max-width:100%` 或强制断词。
- 修正通用 Flex 风险判定：纵向 Flex 不再用横向子元素宽度相加来误判；绝对定位装饰不计入横向宽度；仅在多个不可收缩子项确实超宽时才做预防性换行。
- 旧消息曾被“排版不适配／内容显示不全”维修改丑时，再次执行该维修会先清除旧通用标记，再按剖面保形路线重新安装，恢复原有色块宽度、留白、连接关系和纵向结构。
- 全链路诊断手机端章节新增“剖面／分层保形”计数；此类结构不再被误报为普通 Flex 溢出，因此自动维修会优先指向真实的交互故障。
- 未修改母本库、Prompt、配色或展现形式规则；本次仅修当前 DOM 的交互与手机端工程兼容。


## 0.33.62 修复

- 新增“含脚本镜面静态主体恢复”：当前消息原始兔子镜含 `<script>`、但宿主清洗后只剩空容器时，不再因为发现一个脚本而放弃整面镜子。
- 恢复前先在字符串层把脚本隔离成不可执行占位符，避免脚本正文中的 `<` 运算符被 HTML 紧凑器误当成标签；进入离线 DOM 后彻底删除占位符和残余 `<script>`，绝不执行模型生成的 JavaScript。
- 静态 HTML、`<style>`、checkbox/radio、label、正文、论坛、场景容器与原有 CSS 状态链继续保留；重建后重新执行当前镜面的 ID 作用域与 checked 交互恢复。
- 对脚本中可严格识别的纯装饰生成模式，仅安全解析 `querySelector(.class)`、`createElement('div')`、固定 `className` 与 `appendChild` 关系；星点和城市剪影改为数量受限、位置确定的静态 DOM，不运行随机脚本。
- 仍整面拒绝 `iframe`、`object`、`embed`、`link`、`meta`、`base`、`javascript:` URL 与内联样式中的 `javascript:`；全部 `on*` 内联事件继续删除。
- 存在思维包裹时仍只按当前 summary 隔离这一面兔子镜，不回退到整条消息重绘，不写回 `mes`、`swipe` 或 `display_text`。
- 维修结果会明确说明“脚本已移除，静态主体已保留”；脚本未能安全转译的非装饰逻辑不会被猜测或执行。

## 0.33.61 保留修复

- 新增“兔子镜构思作用域”：母本主题与展现形式仍可参与兔子镜的内容选取、媒介判断、视觉、DOM/CSS 和交互构思，但不得反向进入主回复正文的变量引入、剧情规划、人物行动、角色语言或正文文风。
- 主题条目的旧提示“自然融入本轮剧情气味”已删除，改为仅供兔子镜内部取材、氛围转译、视觉隐喻与媒介细节；不删除“袁立文学”等母本条目，也不禁止模型在兔子镜 UI 构思阶段分析它们。
- “发散孵化”的“外延剧情”改为“兔子镜内部叙事细节”，明确不得为了适配抽取结果而新增或改写主回复剧情。
- 本次没有增加思维链扫描、自动删除或伪装成权限沙箱的监察器；这是生成 Prompt 的用途边界，不会妨碍兔子镜审查正文素材并构思 UI。
- 新增 radio 同组恢复：宿主移除 `name` 后，会从同一条消息的原始兔子镜按 radio 顺序恢复原组，并加当前镜面的唯一前缀，避免单选节点逐个累积为“全部 checked”，也避免不同消息之间互相取消。
- 当多个 radio 已经被错误地同时选中时，只保留当前焦点项；无法判断焦点时保留最后一个当前选中项。之后点击任意节点，始终只有一个分支保持选中。
- 新增 label 代理 checked 路线：模型把 input 包在 `label` 内，却误写 `input:checked ~ .node-content` 时，维修兔会仅在该 label 的局部父容器中，把 label 后方内容作为原 input 的后续兄弟恢复；时间轴节点的正文与描述可随单选状态正确切换。
- 全链路诊断新增 `radio同组恢复` 计数；维修前可报告“radio 同组关系被宿主移除，单选节点会累积成同时选中”。
- 不新增固定分页、滚动框、返回按钮或时间轴模板，不改变母本库抽取概率、配色与视觉规则。

## 0.33.60 保留修复

- 新增“单向 checked 状态”专项巡逻：当 checkbox 进入下一层后把唯一 label／触发器隐藏，同时只留下普通结果正文时，维修兔会识别为缺少返回路线。
- 只对高置信结构安装回退：原始 checkbox 必须不可见、所有关联 label 都在 checked 分支中被隐藏，并且同一分支确实揭示了承担正文或详情的结果层。
- 已展开结果可再次点按返回上一层；取消 checked 后会撤回维修兔写入的 `display:none/block !important`，恢复原触发器并重新隐藏结果。
- 三个或更多独立 checkbox 分支分别维护自己的状态，返回其中一层不会关闭其他已展开分支。
- 结果中已有按钮、链接、输入框等真实交互时不会拦截这些子操作；用户正在选择文字时也不会触发返回。
- radio 组不纳入新的单向结构自动识别，避免擅自清空本应保持单选的状态；旧的已验证可逆路线仍保持原行为。
- 全链路诊断新增“单向checked回退”计数，并在维修前明确报告“checked 分支隐藏了唯一触发器，进入下一层后没有返回上一层的操作入口”。
- 未修改 Prompt、展现形式规则与配色；这是当前兔子镜 DOM 的交互工程维修，不会诱导后续生成固定的返回按钮模板。

- 手机端会将承载这类结果的并排 `details` 改为纵向排列，确保“押注结果／赛后彩蛋”等主要反馈完整显示；背景、配色、边框、阴影和按钮风格保持原样。

- 手机排版巡逻忽略无文字、无子节点、低透明度的绝对定位装饰图形，避免把越界圆环等装饰误报为内容横向溢出。

## 0.33.55 修复

- 新增受限命名函数交互恢复：当模型使用 `onclick="functionName()"`，而宿主同时删除内联事件与 `<script>` 时，维修兔会从当前消息原始兔子镜回读对应函数。
- 仅接受 `document.getElementById(...)`、`classList.contains(...)` 与 `classList.add/remove(...)` 组成的无参数双分支状态机；不执行原始 JavaScript，也不接受网络请求、动态表达式或其他未知语句。
- 自动映射 SillyTavern 重写后的局部 ID 与 `custom-` 类名前缀，可恢复“点击画布／提示文字切换同一组状态层”的结构。
- 同一命名函数的多个触发入口共享同一状态，点击画布、触摸提示与恢复提示均可推进或撤回，不会分别维护互相冲突的状态。
- 维修后将安全状态程序纳入实际恢复计数；当原始 3 个点击入口均已重建时，不再继续报告“宿主删除了会推进状态的点击程序”。
- 维修兔菜单继续使用短摘要，详细证据只保留在巡逻结果与全链路诊断中。

## 0.33.54 保留修复

- 跨父层 `checkbox/radio :checked` 兜底安装后，会立即同步当前已经勾选的初始分支，不再要求用户重新点击一次才能显示结果。
- 同步采用“先撤回旧分支、再应用当前 checked 分支”的方式，避免 radio／checkbox 切换后上一分支的内联急救样式残留。
- 对 `input/change` 和宿主延迟重绘增加分阶段复核，当前状态会在 0／80／260／650ms 再次确认。
- 新增“无可达内容交互”巡逻：当兔子镜只有静态内容或 CSS 动画，且没有控件、按钮、链接、嵌套 details、状态规则或已安装交互路线时，维修兔会明确提示原始输出缺少交互结构。
- 无交互作品只进入准确诊断，不会由维修兔擅自编造按钮、分支、剧情结果或角色反应。
- 全链路诊断新增可达内容交互计数，分别显示交互元素数量、已安装路线数量及是否完全缺失。

## 0.33.53 保留修复

- 小小维修兔将原“文字被裁切或显示不全”入口合并为“📱 排版不适配／内容显示不全”，不再增加重复的手机端按钮。
- 合并路线按“手机端容器关系 → 剩余文字裁切”执行：先处理固定宽度、Grid/Flex 挤压、多栏正文、超宽媒体与横向溢出，再复测叶级文字是否仍被限行、省略或裁切。
- 手机端补丁全部放在当前兔子镜的作用域 `@media (max-width: 640px)` 内；电脑端原布局、背景、配色、渐变、阴影、边框和材质不受影响。
- 两栏文章和文字密集的固定列 Grid 可在窄屏切为单栏；横向时间轴、赛道、表格、画布等媒介保留横向结构，改为当前区域内触摸滚动，不强制改造成纵向卡片。
- 不换行 Flex 会依据子项实际宽度选择换行或纵向排列；固定宽度、`min-width`、`100vw + padding`、长连续字符和超宽图片／SVG／iframe 会被限制在手机容器内。
- 对 `checkbox/radio :checked` 后才展开、但结果区被固定 `height/max-height` 截断的内容，维修兔会保留默认隐藏状态，只在对应控件选中后解除定高裁切。
- 新增手机端排版巡逻 finding 与全链路诊断第 9 节，可分别报告横向溢出、固定宽度、Grid、Flex、多栏、媒体和交互状态内容风险。
- 合并维修只处理用户点击的当前兔子镜，不顺带重排同一消息中的其他镜面。

## 维修边界

- 仅修改当前消息、当前兔子镜的显示 DOM。
- 不写回 `mes`、`swipe` 或 `display_text`。
- 不恢复旧的全局自动急救链。
- DOM 维修仍只处理当前消息、当前兔子镜；生成侧仅收紧母本元素的用途边界，不增加固定 UI 解法或改变抽取库。
- 不修改背景配色、渐变、材质、阴影和装饰主题；只有装饰本身超宽或挡住内容时才限制其尺寸或触摸行为。
- 不把所有横向媒介、固定视窗、弹层、翻面和空间式交互统一改成长页面。

- 多问题发现、分阶段维修、逐步重检和维修后 findings 验证。
- 挨打猫 v1.3、小小维修兔 v1.47、Menu QR v2.1。
