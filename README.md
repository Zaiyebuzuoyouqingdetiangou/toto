# 兔子镜小剧场 / Rabbit Mirror Theater

v0.31.55：基于 v0.31.54，恢复 7.10 的 CSS 动态视觉路径，修复动态模式被引导为 SVG SMIL 后在酒馆中不播放的问题。
本版定点修复：
- 动态视觉重新明确为 CSS 视觉画面，必须使用 `animation + @keyframes` 产生打开即自动持续、肉眼可见的动效。
- `SVG animate` / `animateTransform` 可保留为装饰，但不能再作为完成动态要求的唯一机制。
- 不再接受静态渐变、transition、hover、点击变化代替自动动态。
- 0.31.54 的两次实际输出均只生成 SVG SMIL；本版撤销该错误引导，回到 7.10 稳定路线。
- 动态规则正文由 308 字符降为 305 字符，不增加 Prompt。
- 交互修复、背景包裹、代码块急救、ID 作用域和 7.10 已有 UI 均未改动。

## 既有修复记录

v0.31.54：基于 v0.31.53，修正“动态视觉已勾选但只生成静态渐变”的模式语义。
本版定点修复：
- “动态视觉模式”开启时，必须至少出现一处打开后立即自动、持续运行的真实 CSS/SVG 动画。
- 仅静态渐变、transition、hover 或点击后的状态变化，不再被视为“会动”。
- 动态要求与可交互、发散孵化可以同时成立；三者不再由模型自行择一满足。
- 保留 7.10 的自由取景、视觉画布与氛围审美，不改背景容器和既有 UI。
- 没有新增 Prompt 段落；压缩并替换原动态段落，三开模式的注入字符数低于 v0.31.53。

- 修复模型将初始隐藏状态写在内联 style 后，`:checked` CSS 因优先级不足而完全失效的问题；仅对兔子镜内部的交互状态规则自动追加 `!important`。
- 修复流式输出时 input/label 先出现、末尾 `<style>` 后出现，导致 CSS 仍引用旧 ID、点击后内容不展开的问题。
- 为每个兔子镜保留稳定的 ID 映射，后到的 style、label、ARIA、SVG 引用会继续同步。
- 已处理过的 ID 不会重复加前缀；不修改 Prompt、背景、边框、布局和既有 UI。

- Prompt、主题池、展现形式池与 7.10 审美底盘均未修改，不增加注入内容。
- 代码块急救恢复为“只替换真正代码壳”的逻辑，不再因 `<pre>` 自带背景或 padding 而拒绝急救，也绝不向上替换普通主容器。
- 主视觉背景容器、边框、圆角、阴影、内边距和布局保持原位，急救后正文仍留在原容器内部。
- 交互 label 点击兜底改为捕获阶段，降低主题或其他插件拦截事件后出现“点不开”的概率。
- 每个兔子镜继续独立作用域化 `id`、`for` 与 radio `name`，并补齐 SVG `url(#id)`、ARIA、锚点及 SMIL 引用同步，避免修交互时破坏已有图形和 UI。
- 交互修复仅扫描 `data-rabbit-mirror`（兼容旧属性）的兔子镜，不再处理聊天区内其他普通 `<toto>`。

- 保留 7.10 审美底盘，不沿用 0.31.38～0.31.43 的重度后处理线。
- 删除展现形式池里的「10.1 可交互小剧场」。
- 新增「每轮可交互模式」开关：关闭时不强制内部交互，开启时才每轮强制真实交互。
- 动态渐变模式规则保持原逻辑，不与可交互模式绑定。
- 加回 H.0～H.6 悬疑/怪谈主题组。
- 强化可见中文规则：界面标题、正文、按钮、状态、提示、反馈文案必须使用简体中文。
- 旧 data 属性在急救清洗时会归一为 data-rabbit-mirror。
- 新增主视觉容器完整包裹提示，减少背景空壳、正文掉出主容器的问题。
