# RabbitMirror 0.31.88

基于 0.31.87，仅调整默认交互生成规则。

## 本次修改

- 最外层兔子镜折叠不计入内部交互；每轮内部仍必须至少有一个真实可操作或可探索入口。
- 交互必须从本轮展现形式的结构、阅读方式、空间关系或叙事逻辑中推导。
- 触发后必须产生肉眼可见且有内容意义的变化，假按钮或仅改变边框/颜色/阴影不算有效交互。
- 每轮重新设计交互，减少连续 details/summary 和机械折叠堆叠。
- 禁止内联 JavaScript 事件；hover 不得作为移动端唯一触发方式；交互区域不得被装饰层遮挡。
- UI 自查中的交互检查改为“真实可触发且触发后有明显内容变化”。
